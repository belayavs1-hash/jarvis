# Скилл: /execute-plan

## Назначение

Пройти по утверждённому плану из `_progress.md` шаг за шагом. Самостоятельно исправлять ошибки. Не дёргать пользователя по мелочам — только при блокерах.

## Когда использовать

План в `_progress.md` одобрен пользователем. Пора делать.

## Как работает

### Шаг 0 — Загрузи контекст

Прочитай:
1. `_progress.md` — текущий план и статус задач
2. `git log --oneline -10` — что уже сделано
3. `git status` — текущее состояние файлов

Найди первую незакрытую задачу (`[ ]`) и начни с неё.

### Шаг 1 — Выполни задачу

Для каждой задачи:
1. Прочитай задачу, файл, действие, проверку
2. Выполни действие
3. Запусти проверку
4. Если проверка прошла — отметь задачу выполненной (`[x]`) в `_progress.md`
5. Сделай git commit: `[agent] step N.M: краткое описание`

### Шаг 2 — Ошибки

Если что-то пошло не так:
1. Прочитай ошибку внимательно
2. Попробуй исправить самостоятельно (до 3 попыток)
3. После каждой попытки — снова запусти проверку
4. Если после 3 попыток не получилось — **остановись** и сообщи пользователю:
   - Что делал
   - Какая ошибка
   - Что пробовал
   - Нужна помощь

Не замалчивай ошибки. Не делай вид что всё хорошо.

### Шаг 3 — Переход к следующей задаче

После успешной задачи:
1. Обнови `_progress.md` — перенеси задачу в «Завершено» с датой
2. Сделай короткий коммит
3. Сообщи пользователю: «Готово: [что сделал]. Следующее: [что дальше]»
4. Жди ответа или переходи к следующей задаче (в зависимости от размера шага)

### Шаг 4 — Завершение

Когда все задачи закрыты:
1. Запусти финальную проверку всего что сделано
2. Сообщи пользователю о завершении
3. Удали `_progress.md`
4. Предложи следующий шаг исходя из `GOALS.md`

## Когда останавливаться и спрашивать

Стоп и вопрос пользователю если:
- Ошибка не исправляется после 3 попыток
- Следующий шаг требует данных которых нет (ключи API, пароли, выбор из вариантов)
- Задача противоречит тому что уже сделано
- Нужно удалить файлы или данные (красная зона)

По мелочам (синтаксические ошибки, не тот отступ, опечатка в пути) — исправляй сам, не спрашивай.

## Важные правила

- **Один шаг за раз** — не перепрыгивай задачи
- **Коммит после каждого шага** — каждый коммит = рабочее состояние
- **Обновляй `_progress.md`** после каждого выполненного шага
- **Не переписывай файлы целиком** если можно точечно поправить
- **Проверяй реально**, не говори «сделано» не убедившись
